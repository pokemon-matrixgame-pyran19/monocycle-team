# rotating-team ツール 使い方

「相手構築を θ だけ回したときのゲームの値 `g(θ)`」を計算・描画するための小道具。
**計算コアは1つ（`model.js`）だけ**で、それを CLI（`cli.js`）と描画 HTML（`experient.html`）が共有する。
記事の数式・グラフ読み取りを検算するときは、Python などで毎回モデルを書き直さず、ここを使う。

```
tools/rotating-team/
├── model.js        計算コア（純粋関数のみ。ブラウザ／Node 両対応）
├── cli.js          CLI から g(θ) を読み取る
├── experient.html  スライダー付きインタラクティブ・グラフ（model.js を読み込む）
└── docs/README.md  このファイル
```

前提: Node.js（`node --version` が通ればよい。確認時は v22 系）。インストール不要・依存パッケージなし。

---

## パラメータ

すべて記事の回転パートと同じ記号。角度は **度**。

| キー | 意味 | 既定値 |
|---|---|---|
| `g1`,`g2` | 自分の相性ベクトル `v1,v2` の偏角 γ₁,γ₂ | 60, 120 |
| `d3`,`d4` | 相手の `v3,v4` の偏角 δ₃,δ₄（θ=0 基準） | 40, 140 |
| `r1`..`r4` | 各ベクトルの長さ `|v1|..|v4|` | 1,1,1,1 |

θ を動かすと相手側（`v3,v4`）だけが回り、自分側は固定。既定値は記事のサンプル配置。

---

## CLI（`cli.js`）

```bash
cd tools/rotating-team

# 既定（記事のサンプル配置）でまとめを表示
node cli.js

# パラメータ上書き
node cli.js --g1 5 --g2 175 --d3 40 --d4 140

# 指定 θ での g と領域（カンマ区切り可。--at でも可）
node cli.js --theta 60,90,120,220

# 長さを変える（相性の影響の受けやすさ）
node cli.js --r1 2 --r2 2 --r3 2 --r4 2

# θ,g,領域 を CSV で一覧（プロット用データの吐き出し）
node cli.js --table 1 > curve.csv

# ヘルプ
node cli.js --help
```

### 既定出力の読み方

```
params: γ1=60 γ2=120 δ3=40 δ4=140  r=(1,1,1,1)
|V_i|=1.155  |V_j|=1.556
2択の窓: (60.3°, 119.8°) と (240.3°, 299.8°)
max g  = 0.643 @ 60.0°     ← 最大の有利とその θ
min g  = -0.866 @ 220.0°   ← 最小（最大の不利）
max|g| = 0.866 @ 220.0°    ← 絶対値が一番大きい点
```

- `2択の窓` … 本質的に2択になる θ 区間（実際のレジームから数値検出）。窓の外は実質一択。
- `--step` で窓検出の解像度が変わる（既定 0.5°）。`--table N` は同時に刻みも N に合わせる。

### よくある検算

```bash
node cli.js --r1 2 --r2 2 --r3 2 --r4 2   # 長さを上げると max|g|>1（一択頭打ちが |v_a||v_b| まで上がる）
node cli.js --g1 5 --g2 175               # 広がりを極端にしても max g は跳ねない（R_i,R_j が打ち消す）
```

## ブラウザ（`experient.html`）

`experient.html` と `model.js` を**同じフォルダに置いたまま**開く（HTML は相対パス `model.js` を読む）。
スライダーで γ,δ,長さを動かすと、`g(θ)` 曲線・2択の窓・`|V_i|,|V_j|` がリアルタイムに更新される。
別ページに埋め込むときは2ファイルを同じ階層に置くか、`<script src>` を実際の配置に合わせて直す。

---

## Node から直接使う（`model.js` API）

```js
const RT = require('./model.js');
const p = { ...RT.DEFAULTS, g1: 60, g2: 120, d3: 40, d4: 140, r1: 1, r2: 1, r3: 1, r4: 1 };

RT.gval(90, p);     // → { g, two, a, b, c, d }   two=true なら本質的に2択
RT.radii(p);        // → { Ri, Rj }
RT.sample(p, 0.5);  // → [{ th, g, two }, ...]（θ=0..360）
RT.bands(pts);      // → [[start,end], ...] 2択の窓
RT.summary(p);      // → { Ri, Rj, bands, max, min, maxAbs, pts }
```

| 関数 | 役割 |
|---|---|
| `vecs(th, p)` | θ での4ベクトル `{v1,v2,v3,v4}` |
| `Rof(va, vb)` | 構築ベクトルの大きさ `|v_a - v_b| / |v_a × v_b|` |
| `gval(th, p)` | その θ のゲームの値と領域（鞍点 or 混合） |
| `radii(p)` | `|V_i|, |V_j|` |
| `sample(p, step)` | θ=0..360 のサンプル列 |
| `bands(pts)` | 2択の窓を数値検出 |
| `summary(p, step)` | 上記をまとめて＋最大/最小/最大絶対値 |

`model.js` は UMD ラッパなので、Node では `require`、ブラウザでは `window.RotatingTeam` として同じものが使える。**計算ロジックを書き足すときは必ず `model.js` 側に入れる**（CLI と HTML が自動で共有され、二重実装＝食い違いを防げる）。
