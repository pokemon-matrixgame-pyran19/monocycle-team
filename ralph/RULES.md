# 規約（Ralph ループ用）

## 作業単位

- 1イテレーション = 1タスク。大きなタスクは PLAN.md 上で分割してから着手する。
- 迷ったら小さい方を選ぶ。1つの .md ファイルに閉じる変更が理想。
- 人間の判断が必要な事項（理論の方向転換、既存結果と矛盾する発見など）は勝手に決めず、PLAN.md の「要判断」に書いて別のタスクに移る。

## 配置と命名

- 研究成果は `theory/theory/02-power_difference/` に置く（初回イテレーションで作る）。
- theory/README.md の規約に従う：ディレクトリは連番2桁、トピック名は英小文字とアンダースコア。
- 下書き・途中計算・没案は `theory/idea/` に置く。`theory/theory/` には確定した論理だけを置き、書きかけを残さない。
- `theory/manuscript/` には触れない（投稿用下書きはループの対象外）。
- 00・01 の既存ファイルは原則書き換えない。例外は 02 への参照リンクの追記など軽微なもののみ。
- ralph/ 内でエージェントが書き換えてよいのは PLAN.md と JOURNAL.md だけ。

## 執筆スタイル

- 既存トピック（00・01）の形式に合わせる：
  - トピック直下に README.md（概要・ファイル一覧・要点）を置き、個別の内容は独立した .md に分ける。
  - 数式は LaTeX（`$...$`、`$$...$$`）。
  - 文体は既存ファイルと同じ「だ・である」調の簡潔な説明文。
- 記号は既存の定義を引き継ぐ：パワー $p$、相性ベクトル $v=(x,y)$、外積 $v_i\times v_j$、構築ベクトル $V$、長さの和 $S$、広がり効率 $\eta$、ゲームの値 $g$。
- 新しい記号を導入するときは初出で必ず定義し、02 の README にも記号の一覧として載せる。
- 結論には導出を付ける。導出が長くなる場合はファイルを分け、README から流れが追えるようにする。

## 検証

- 閉じた式を出したら、具体的な数値例を最低1つ入れて検算する。
- **等パワー極限（パワー差 → 0）で 00・01 の結果に戻ること**を必ず確認し、本文にも明記する。等パワー側の数値は既存の `tools/rotating-team/`（cli.js）で出せる。
- `tools/rotating-team/` は等パワー専用のモデルなので書き換えない。パワー差込みの検算をリポジトリに残すときは `tools/power-difference/` を新設する（rotating-team の流儀に合わせて依存なしの Node.js、model と cli を分ける）。
- 使い捨ての計算はコミットしなくてよい。残す価値のある検算だけを tools/ に置く。

## Git

- コミットメッセージ：
  - 理論本体の変更 → `theory/02: <やったこと>`
  - ループ運用ファイル（PLAN / JOURNAL）だけの変更 → `ralph: <やったこと>`
- 動く状態・読める状態でコミットする。theory/theory/ に書きかけを残す状態でイテレーションを終えない（続きものは idea/ に退避して次回拾う）。
