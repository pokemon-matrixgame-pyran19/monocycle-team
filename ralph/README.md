# Ralph ループ（自律研究ループ）

同じプロンプトを繰り返し与えるだけで研究が前に進む仕組み。エージェントの記憶には頼らず、状態（目標・バックログ・進捗）をすべてこのディレクトリのファイルとして持つ。

現在の目標：theory/theory/ に **02（パワー差がある場合の理論）** を組み上げる → [GOAL.md](GOAL.md)

## ファイル構成

| ファイル | 役割 | 書き換える人 |
|---|---|---|
| [PROMPT.md](PROMPT.md) | 毎回の実行手順（ループ本体） | 人間 |
| [RULES.md](RULES.md) | 研究・執筆・Git の規約 | 人間 |
| [GOAL.md](GOAL.md) | 研究目標と完了条件 | 人間 |
| [PLAN.md](PLAN.md) | タスクバックログ | エージェント |
| [JOURNAL.md](JOURNAL.md) | イテレーションごとの記録 | エージェント |

「人間」のファイルはループの外から方向を制御するためのもの。エージェントは PLAN.md / JOURNAL.md 以外の ralph/ 内ファイルを書き換えない。

## セットアップ（ウェブ版 Claude Code のデイリースケジュール）

スケジュールのプロンプトに以下を貼る：

> ralph/PROMPT.md を読んで、その手順に従って自律研究ループを1イテレーション実行してください。

プロンプト本体はリポジトリ内（PROMPT.md）にあるので、指示を変えたいときはスケジュールを触らずファイルを編集すればよい。

## 運用上の注意

- **PR のマージがループの前提**。各実行は main から始まるため、前回の PR をマージしないと PLAN / JOURNAL の更新が次回に見えず、同じ作業を繰り返しうる。次の実行までに1回マージする（内容を却下する場合も、JOURNAL のエントリだけ拾って main に入れると記録が途切れない）。
- 方向を変えたいときは GOAL.md か PLAN.md を直接編集する。エージェントへの指示は全部ファイル経由で行う。
- PLAN.md の「要判断」に質問が溜まったら、回答を同じ場所に書き込む（次のイテレーションが拾う）。
- **終了**：GOAL.md の完了条件を満たすと、以後のイテレーションは校閲・保守モードに落ちる（PROMPT.md 手順2）。ほぼ何もしない実行が続くようになったらスケジュールを止めてよい。判定は緩めに設計してある。

## 由来

Ralph loop（Geoffrey Huntley）：同一プロンプトをエージェントに繰り返し与え、状態管理をリポジトリ内のファイルに外部化する手法。ここでは連続ループの代わりにデイリースケジュールで駆動するので厳密には Ralph ループそのものではないが、「状態はファイル、プロンプトは不変、1回1タスク」という骨格は同じ。
