# Ralph ループ実行プロンプト

あなたはこのリポジトリで**自律研究ループの1イテレーション**を実行する。状態はすべてファイルにある。以下の手順に上から順に従うこと。

## 原則

- **1回の実行で進めるタスクは1つだけ**。小さく確実に前進し、続きは次回に回す。
- 記憶や推測に頼らない。判断の根拠は必ずファイル（GOAL / RULES / PLAN / JOURNAL / theory 本体）から取る。
- 作業の痕跡（PLAN・JOURNAL の更新）を残さずに終了しない。次のイテレーションはこの2ファイルと git log しか見ない。

## 手順

### 1. 状況把握（必ず読む）

1. [GOAL.md](GOAL.md) — 研究の目標と完了条件
2. [RULES.md](RULES.md) — 規約
3. [PLAN.md](PLAN.md) — タスクバックログ
4. [JOURNAL.md](JOURNAL.md) — 直近のイテレーション記録（新しい方から2〜3件でよい）
5. `theory/theory/02-power_difference/` — 現状の成果物（まだ無ければ 00・01 の README を読む）
6. `git log --oneline -15` — 直近のコミット

### 2. 完了判定

GOAL.md の「完了条件」をすべて満たしているか確認する。

- **満たしている場合**：新しい研究はしない。02 の校閲パス（誤字・数式の整合・リンク切れ・検算の再実行）を1回だけ行う。直すものが無ければ JOURNAL.md に「完了状態を確認、作業なし」とだけ追記して手順6へ。
- 満たしていない場合：次へ。

### 3. タスク選択

- PLAN.md の「必須」の未完了タスクから、他のタスクの土台になるものを優先して**1つ**選ぶ。必須が尽きていれば「発展」から選ぶ。
- 「要判断」に人間の回答が書き込まれていれば、その反映を最優先タスクとする。
- JOURNAL.md と git log を見て、直前の実行と重複しないことを確認する。
- 選んだタスクが1イテレーションに大きすぎるなら、PLAN.md 上で分割してから最初の一片だけをやる。

### 4. 実行

- RULES.md の規約（配置・記法・検証）に従って研究・執筆する。
- 試行錯誤や途中計算は theory/idea/ に置き、確定した論理だけを theory/theory/02-power_difference/ に置く。
- 閉じた式を出したら数値検算し、等パワー極限（パワー差 → 0）で 00・01 の結果に戻ることを確認する。
- 行き詰まったら無理に結論を出さない。詰まった内容と試したことを JOURNAL.md に書いて次回に回す。人間の判断が要る事項は PLAN.md の「要判断」に書く。

### 5. 記録

- PLAN.md：完了したタスクにチェックを入れ、作業中に見つかった新タスクを適切な節に追記する。
- JOURNAL.md：冒頭のフォーマットに従い、今回のエントリを**一番上に**追記する。

### 6. コミットと終了

- 変更をコミットしてプッシュする（メッセージ規約は RULES.md）。ブランチや PR の作成は実行環境の標準の流儀に従ってよい。
- 最後に、今回やったことと次回の推奨タスクを1〜3行で報告して終了する。
